如何将hexo部署到github page上

视频链接 提取码
https://pan.baidu.com/s/1sK2dH8I9rz1vAmu9ApP1xg 提取码: uvjf

配合视频观看效果更好

正文

  1. 点击start project 或者下面的new repository 创建一个新的仓库
  2. Github仅能用一个同名仓库的代码托管一个静态站点
  3. 然后打开仓库创建一个 index.html 文件
  4. 后面把hexo的文件部署就可以在网站上看到博客
  5. git config –global user.name 你的GitHub用户名
  6. git config –global user.email 你的GitHub注册邮箱
  7. ssh-keygen -t rsa -C github注册邮箱
  8. 按三次回车
  9. 电脑/用户/用户名/.ssh 文件夹 用记事本打开
  10. 打开github设置keys （如何在仓库的设置不成功，请点击github的头像里面的设置，左边的栏目有一个SSH and GPG keys 请在此处设置，和视频中设置方法一致）
  11. 设置完输入ssh git@github.com判断是否成功
  12. 这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。
  13. 打开站点配置文件config 填写自己创建的github仓库的完整路径 记得加上.git（示例：https://github.com/用户名/用户名.github.io.git）
  14. 输入命令npm install hexo-deployer-git –save
  15. Hexo clean hexo g hexo d

视频最后由于一些原因，网页地址不是用户名.github.io，如果自己创建，直接使用用户名.github.io进入博客即可
